Goal

One cross-tool scaffold skill (user-invocable: true) with two modes. Setup mode (scaffold setup, run once in the source repo) walks the repo and writes docs/scaffold/spec.yaml + templates/ describing the project's structure, conventions, context-tree nodes, bundled skills, and variables. Seed mode (scaffold <target-path>) reads the spec and generates a new plugin project at the target — root files, context-tree skeleton, bundled skills, manifests — with {{ var }} placeholders substituted. Pattern from mattpocock/skills, folded into a single skill per the user's call. A skill (not a command) so Codex and any agentskills.io runtime can run it.

Context

Every new plugin project repeats the same setup: AGENTS.md/CLAUDE.md pair convention, context-tree nodes, plugin manifests, bridge-skill wiring, starter skills. Repeat work + drift-prone. Scaffolding from a single versioned spec keeps every new project starting at the same baseline.

Depends on:

  • foundation-categorization-scoring — spec references category paths (engineering/productivity)
  • tree-skill (context-tree) — seed mode writes context-tree-shaped projects; the node convention IS the scaffold output
  • pipelines-to-skills — established the commands-removed, skills-only rule this plan follows
